## 三、 共享 MySQL / Redis 环境下的代码配置

为了做到“既能共享一套环境，又互不干扰”，配置文件的管理至关重要：

### 1. 动态激活环境：`application.yml`

主配置文件只负责环境的激活和公共配置：

YAML

```
spring:
  profiles:
    active: dev # 团队统一开发环境
```

### 2. 团队共享配置：`application-dev.yml`

在这个文件里配置你们**共享的远程 MySQL 和 Redis**。

- **警告：** 如果这个文件包含真实密码且仓库是公开的，请不要直接推送到公共仓库。可以使用环境变量，或者在本地使用 `application-local.yml` 覆盖。
- **MyBatis-Plus 配置：**

YAML

```
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://119.xxx.xxx.xxx:3306/cloud_hospital?useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: your_shared_password
  data:
    redis:
      host: 119.xxx.xxx.xxx
      port: 6379
      password: your_redis_password

mybatis-plus:
  configuration:
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl # 开发阶段开启控制台 SQL 打印
  global-config:
    db-config:
      id-type: auto # 团队共用库，主键自增策略最稳妥，避免雪花算法在并发时因机器ID重复导致冲突
```
