# 前端登录鉴权对接指南（v2 / PR1～PR5 对齐）

本文档说明前端如何与后端 JWT v2 账号化认证链路对接，涵盖三角色路由、HTTP 状态处理、患者注册、账号管理与 Token 全局失效。

---

## 一、核心约定

### 1.1 Token 传输

后端生成裸 JWT。前端在请求受保护接口时拼接 Bearer 前缀：

```http
Authorization: Bearer <裸token>
```

- 请求头名固定为 **`Authorization`**
- 值以 `Bearer ` 开头（注意空格）
- 裸 token 存本地，发送时拼接，不把前缀存入 token

### 1.2 角色来源（重要）

```text
role 只能来自后端登录响应 data.userInfo.role 或 GET /api/v1/auth/me。
前端不得从登录页下拉框、localStorage 手工设置或 URL 参数读取角色。
X-Role 请求头没有权限效力，后端不读取 X-Role。
```

### 1.3 Token 中的敏感信息

JWT claims 含 `ver=2`、`role`、`tv`（token_version）、`employeeId`、`patientId`。
以上均由后端签名，前端**不得解析、修改或自行构造** JWT。需要角色/身份信息时调 `/auth/me`。

### 1.4 401 后自动重登

后端 PR5 引入 `token_version`：管理员禁用账号、重置密码或用户自行改密码后，该账号**所有历史 Token 立即 401**。前端 401 处理必须清除本地 token 并跳转登录页，不可尝试重试或恢复旧 token。

## 二、HTTP 状态码处理

| HTTP | Result.code | 含义 | 前端处理 |
|------|------------|------|----------|
| 200  | 200 | 正常 | 正常处理 |
| 200  | 400 | 业务参数/校验错误 | 读取 `message` 展示 |
| 200  | 409 | 操作冲突（重复绑定等） | 读取 `message` 展示 |
| 200  | 404 | 旧版资源不存在（部分旧接口） | 按 body.code 判断 |
| 401  | 401 | 未登录/Token 过期/无效/tv 不匹配 | **清空 token + userInfo，跳转登录页** |
| 403  | 403 | 权限不足（角色不允许） | 展示"无权访问"，**不清空 token，不跳转登录页** |
| 404  | 404 | 资源不存在或无权查看 | 展示"资源不存在" |
| 423  | 423 | 账号临时锁定 | 展示"账号已锁定，请稍后再试" |

> **关键**：401 vs 403 必须区分处理。401 = 凭证问题 → 重新登录。403 = 当前角色无权限 → 仅提示，不能跳登录（否则会死循环）。

### 响应拦截器示例

```js
http.interceptors.response.use(
  res => {
    const httpStatus = res.status
    const { code, message, data } = res.data
    if (httpStatus === 401 || code === 401) {
      localStorage.removeItem('token')
      localStorage.removeItem('userInfo')
      router.push('/login')
      return Promise.reject(new Error(message || '登录已失效'))
    }
    if (httpStatus === 403 || code === 403) {
      return Promise.reject(new Error(message || '权限不足'))
    }
    if (code !== 200) {
      return Promise.reject(new Error(message))
    }
    return data
  },
  err => {
    if (err.response && err.response.status === 401) {
      localStorage.removeItem('token')
      localStorage.removeItem('userInfo')
      router.push('/login')
    }
    return Promise.reject(err)
  }
)
```

## 三、登录接口

```
POST /api/v1/auth/login     ← @Public，无需 Token
Content-Type: application/json

请求体：
{
  "username": "doctor01",
  "password": "<PASSWORD>"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `username` | String | 是 | 登录账号（user_account.username，不是员工 ID） |
| `password` | String | 是 | 密码（明文传输，后端 BCrypt/MD5 校验） |

### 成功响应（HTTP 200）

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "expiresAt": 1782216675000,
    "userInfo": {
      "accountId": 2,
      "role": "DOCTOR",
      "employeeId": 1,
      "patientId": null,
      "id": 1,
      "realname": "测试医生",
      "deptmentId": 1,
      "deptName": "神经内科",
      "registLevelId": 1,
      "registLevelName": "普通号"
    }
  }
}
```

| 字段 | 说明 |
|------|------|
| `data.token` | 裸 JWT，前端持久化 |
| `data.expiresAt` | 毫秒时间戳，默认登录 8 小时后过期 |
| `userInfo.role` | **ADMIN / DOCTOR / PATIENT**，前端路由/菜单据此渲染 |
| `userInfo.accountId` | 账号 ID |
| `userInfo.employeeId` | 员工 ID（DOCTOR 有值，PATIENT 为 null） |
| `userInfo.patientId` | 患者 ID（PATIENT 有值，其余为 null） |
| `userInfo.id` | 旧字段，等价 employeeId |
| `userInfo.realname` | 展示姓名 |
| `userInfo.deptName` | DOCTOR 所在科室（PATIENT 为 null） |

**失败：**

| HTTP | code | message | 场景 |
|------|------|---------|------|
| 401 | 401 | 账号或密码错误 | 用户名不存在/密码错/账号已禁用（统一安全消息） |
| 423 | 423 | 账号已锁定，请稍后再试 | 连续失败 ≥5 次，锁定 15 分钟 |

## 四、当前用户信息

```
GET /api/v1/auth/me     ← 需要 Bearer Token，三角色均可
```

返回格式同登录的 `userInfo`（不含 token/expiresAt）。前端可用于恢复会话。

---

## 五、登出

```
POST /api/v1/auth/logout     ← 需要 Bearer Token
```

无请求体。后端将该 token 加入 Redis 黑名单（即使用旧 token 也无法再访问，立即 401）。

登出后前端应清除本地 token + userInfo。

---

## 六、修改密码

```
PUT /api/v1/auth/password     ← 需要 Bearer Token
{
  "oldPassword": "旧密码",
  "newPassword": "新密码（≥6 位）"
}
```

成功后 `token_version` 递增，**当前 Token 及所有历史 Token 立即 401**。前端应提示用户用新密码重新登录。

## 七、三角色页面与接口映射

以下映射仅描述后端实际权限，前端路由守卫按 `userInfo.role` 做体验控制。

### ADMIN

| 模块 | 接口前缀 | 说明 |
|------|---------|------|
| 科室 | `/api/v1/department/**` | 增删改查 |
| 挂号级别 | `/api/v1/regist-level/**` | |
| 结算类别 | `/api/v1/settle-category/**` | |
| 排班 | `/api/v1/scheduling/**` | |
| 员工 | `/api/v1/employee/**` | |
| 药品字典 | `/api/v1/drug-info/**` | 与 DOCTOR 共享 |
| 疾病字典 | `/api/v1/disease/**` | 与 DOCTOR 共享 |
| 医技项目 | `/api/v1/medical-technology/**` | 与 DOCTOR 共享 |
| 挂号 | `/api/v1/register/**` | 跨医生管理（见注） |
| 处方 | `/api/v1/prescription/**` | 跨医生管理 |
| 检查 | `/api/v1/check-request/**` | 跨医生管理 |
| 检验 | `/api/v1/inspection-request/**` | 跨医生管理 |
| 处置 | `/api/v1/disposal-request/**` | 跨医生管理 |
| **账号管理** | `/api/v1/account/**` | 创建/查询/启禁用/重置密码 |

> ADMIN 可跨医生查看和操作所有挂号及其派生记录，不受 `register.employee_id` 限制。

### DOCTOR

| 模块 | 接口前缀 | 说明 |
|------|---------|------|
| 药品字典 | `/api/v1/drug-info/**` | 只读 + 查询 |
| 疾病字典 | `/api/v1/disease/**` | 只读 + 查询 |
| 医技项目 | `/api/v1/medical-technology/**` | 只读 + 查询 |
| 挂号 | `/api/v1/register/**` | **仅本人接诊**（`register.employee_id = 自己`） |
| 处方 | `/api/v1/prescription/**` | **仅本人接诊挂号下的处方** |
| 检查 | `/api/v1/check-request/**` | **仅本人接诊挂号下的检查** |
| 检验 | `/api/v1/inspection-request/**` | **仅本人接诊挂号下的检验** |
| 处置 | `/api/v1/disposal-request/**` | **仅本人接诊挂号下的处置** |

> DOCTOR 访问本人接诊范围外的 register 或派生记录 → **HTTP 404**（不透露记录是否存在）。
> DOCTOR 访问员工/科室/排班/账号管理 → **HTTP 403**。

### PATIENT

| 接口 | 说明 |
|------|------|
| `GET /api/v1/patient/profile` | 个人资料（无身份证/手机号/地址） |
| `GET /api/v1/patient/records` | 就诊记录列表（仅 `register.patient_id = 自己`） |
| `GET /api/v1/patient/records/{registerId}` | 就诊详情（含病历/处方/检查/检验/处置） |
| `POST /api/v1/auth/logout`、`GET /auth/me`、`PUT /auth/password` | 与三角色共用 |

> PATIENT 访问其他患者 registerId → **HTTP 404**。 访问旧通用接口 → **HTTP 403**。

## 八、患者自助注册

```
POST /api/v1/auth/patient/register     ← @Public
Content-Type: application/json

{
  "username": "new_patient",
  "password": "mypass123",
  "realName": "新患者",
  "gender": "男",
  "birthdate": "1990-01-01",
  "cardNumber": "210102199001011234",
  "phone": "13800000000",
  "homeAddress": "沈阳市和平区"
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| `username` | 是 | 3-64 字符，字母/数字/下划线 |
| `password` | 是 | 6-72 字符 |
| `realName` | 是 | 真实姓名 |
| `gender` | 是 | 男/女 |
| `birthdate` | 是 | YYYY-MM-DD |
| `cardNumber` | 是 | 18 位身份证格式（`^[0-9]{17}[0-9Xx]$`，仅格式校验） |
| `phone` | 否 | 手机号 |
| `homeAddress` | 否 | 家庭住址 |

**成功：HTTP 200 + code=200，不返回 JWT。** 前端应提示"注册成功，请登录"，并跳转到登录页。

**失败：** 资料不一致、已绑定账号、用户名重复 → **HTTP 200 + code=400**，`message` 为统一安全提示"注册失败，请核对提交信息或联系管理员"（不泄露具体原因）。

## 九、账号管理 API（仅 ADMIN）

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/v1/account` | 分页查询（role/status/username 筛选） |
| `GET` | `/api/v1/account/{id}` | 详情（无密码/身份证/手机号/地址） |
| `POST` | `/api/v1/account/employee` | 创建 ADMIN/DOCTOR 账号，绑定 employee |
| `POST` | `/api/v1/account/patient` | 创建 PATIENT 账号，绑定 patient |
| `PATCH` | `/api/v1/account/{id}/status` | 启用/禁用（body: `{"status": 0|1}`） |
| `POST` | `/api/v1/account/{id}/reset-password` | 重置密码（body: `{"newPassword": "..."}`） |

> 禁用/重置密码 → 目标账号所有 Token 立即 401。禁止禁用自己或最后一个 ADMIN。
> 创建患者账号响应不含身份证/手机号/地址/密码。
